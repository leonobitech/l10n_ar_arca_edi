Argentina - ARCA Electronic Invoicing

Odoo 19 Community AGPL-3 Beta

Functional

This module integrates Odoo with ARCA (Agencia de Recaudación y Control Aduanero, formerly AFIP) web services for electronic invoicing in Argentina. It handles the full lifecycle of electronic invoice authorization, from digital certificate management to CAE (Código de Autorización Electrónico) requests and verification.

Configuration Steps

  1. Generate Certificate: Go to Settings > Invoicing > ARCA Electronic Invoicing and create a new digital certificate. The module generates RSA keys and CSR directly from Odoo.
  2. Register in ARCA Portal: Download the CSR file and upload it to the ARCA portal (auth.afip.gob.ar) to obtain the signed certificate (.crt).
  3. Authorize Web Service: In the ARCA portal, authorize the wsfe service for your CUIT and the certificate's Distinguished Name (DN).
  4. Upload Certificate: Back in Odoo, upload the signed .crt file and click Test Connection to validate the WSAA authentication.
  5. Configure Journal: Go to your Sales journal and set the ARCA POS System to "Online Invoice" with the corresponding POS number.
Dual Environment Support: The module supports both Testing (homologación) and Production environments. Use the testing environment to validate your setup before going live.

How It Works

  • Automatic CAE: When you confirm (post) an invoice, the module automatically requests a CAE from ARCA. The CAE, expiration date, QR code, and barcode are stored on the invoice.
  • Manual CAE Request: If automatic CAE fails or is disabled, use the "Request CAE" button in the invoice header.
  • Verify CAE: Use the "Verify CAE" button to validate an invoice's CAE against ARCA servers (works with both testing and production).
  • PDF Invoice: The printed invoice includes the ARCA QR code, CAE number, and expiration date in the footer, compliant with RG 4892/2020.
  • Token Caching: WSAA authentication tokens are cached for their full validity period (~12 hours), minimizing API calls.
  • Certificate Monitoring: A scheduled action monitors certificate expiration and sends alerts before they expire.

Technical

This module implements the ARCA electronic invoicing web services using SOAP (via zeep) and CMS/PKCS#7 authentication (via cryptography).

ARCA Web Services

ServiceDescriptionEndpoint
WSAAAuthentication & AuthorizationLogin Ticket Request via CMS signed XML
WSFEv1Electronic Invoice (Domestic)CAE request, invoice query, parameters
WSCDCInvoice VerificationThird-party invoice validation

Models

ModelDescription
l10n_ar.arca.certificateDigital certificate management (RSA keys, CSR, CRT). Handles key generation, CSR creation, certificate upload, and connection testing.
l10n_ar.arca.wsaaWSAA authentication service. Creates CMS-signed Login Ticket Requests, manages token caching with automatic refresh.
l10n_ar.arca.wsfeWSFEv1 invoice service. Handles CAE requests (FECAESolicitar), invoice queries (FECompConsultar), and parameter retrieval.
account.moveExtended with CAE fields, QR code generation (RG 4892/2020), barcode generation, and automatic CAE on invoice confirmation.
account.journalExtended with l10n_ar_arca_edi_enabled flag to enable/disable electronic invoicing per journal.

Regulatory Compliance

  • RG 4892/2020: QR code on printed invoices for digital verification
  • RG 2485: Electronic invoicing general regulations
  • RG 5616: Customer IVA condition reporting (mandatory since April 2025)
